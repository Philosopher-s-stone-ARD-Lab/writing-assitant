# Contributing to Writer's Companion

Thank you for your interest in contributing to Writer's Companion! This document provides guidelines and information for contributors.

## Code of Conduct

By participating in this project, you agree to abide by our code of conduct:

- Be respectful and inclusive
- Welcome newcomers and help them learn
- Focus on what is best for the community
- Show empathy towards other community members

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue on GitHub with the following information:

- A clear and descriptive title
- Steps to reproduce the bug
- Expected behavior vs. actual behavior
- Environment information (OS, browser, Node.js version)
- Any relevant screenshots or error messages

### Suggesting Features

We welcome feature suggestions! Please create an issue with:

- A clear description of the feature
- Why this feature would be useful
- Any implementation ideas you might have

### Development Setup

1. Fork the repository
2. Clone your fork locally
3. Install dependencies: `npm install`
4. Create a new branch: `git checkout -b feature/your-feature-name`
5. Make your changes
6. Test your changes thoroughly
7. Commit your changes: `git commit -m "feat: add your feature"`
8. Push to your fork: `git push origin feature/your-feature-name`
9. Create a pull request

### Coding Standards

- Use TypeScript for all new code
- Follow the existing code style and patterns
- Use meaningful variable and function names
- Add comments for complex logic
- Ensure your code is properly formatted

### Testing

- Test your changes manually in the browser
- Ensure all existing functionality still works
- Test on different screen sizes (responsive design)
- Verify that environment variables are properly handled

### Pull Request Process

When submitting a pull request:

1. Provide a clear and descriptive title
2. Describe your changes in the description
3. Reference any related issues
4. Include screenshots if your changes affect the UI
5. Ensure all checks pass

## Project Structure

Understanding the project structure will help you contribute effectively:

- `components/`: React components
- `services/`: API integration services
- `types.ts`: TypeScript type definitions
- `constants.tsx`: App constants and persona definitions

## Areas for Contribution

We're particularly interested in contributions to:

- UI/UX improvements
- New persona ideas
- Enhanced chat features
- Performance optimizations
- Documentation improvements
- Accessibility improvements

## Getting Help

If you need help:

- Check existing issues and discussions
- Read the documentation in the README
- Ask questions in GitHub discussions
